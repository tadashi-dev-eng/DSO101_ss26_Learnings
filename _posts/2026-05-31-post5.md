## What is Jenkins
- Jenkins is an open-source automation server written in Java that helps automate various aspects of software development, including building, testing, and deploying applications.
- It enables continuous integration and continuous delivery, allowing teams to detect issues early and release software more frequently and reliably.

### Continuous Integration (CI)

- CI involves taking code from a repository, packaging it, and running tests such as unit tests, integration tests, and security checks to ensure the code is ready for deployment.
- The process is likened to wrapping a gift: assembling the parts, verifying everything is correct, and preparing it for delivery.

### Continuous Delivery and Continuous Deployment (CD)

- CD is the process of deploying the packaged code to a target system, which could be serverless, a virtual machine, container, or on-premises infrastructure.
- Continuous delivery requires manual intervention to deploy the code, such as clicking a button in a CI/CD tool, while continuous deployment automates the entire process with no human intervention.

Key Differences Between Continuous Delivery and Continuous Deployment

- Continuous delivery involves a manual step to trigger deployment after CI completes.
- Continuous deployment automatically deploys code immediately after successful CI without any manual action.
- CD processes ensure authentication to the deployment system and verify that the deployed application is functioning correctly through tests.

### Question : What is the main purpose of continuous integration (CI)?
- The main purpose of continuous integration (CI) is to take the code from a repository, package it, and run automated tests (such as unit tests, integration tests, and security checks) to ensure the code is ready and reliable before deployment. This helps catch errors early and ensures that the code is properly assembled and functional.

## Why JENKINS

- Jenkins is completely free and open-source, allowing users to host it on virtual machines, containers, or locally for development.
- Jenkins can be self-hosted, giving users full control over their CI/CD environment.
