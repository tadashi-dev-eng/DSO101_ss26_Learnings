## Building Docker Containers
- A Dockerfile contains instructions to create a Docker image from a base container image.
- Instructions include specifying the base image, running commands, setting environment variables, and more.
- Saving a Dockerfile allows automatic creation of a Docker image by running the Dockerfile.
- The Dockerfile commands act on the base image to create a new image with additional layers.

### Question : What is the role of a Dockerfile in building Docker images?
The Dockerfile plays a central role in building Docker images by providing a set of instructions or commands that define how to create a new image from a base image. 

### Question : What is the purpose of the RUN command in a Dockerfile?
The RUN command in a Dockerfile is used to execute commands inside the image during the build process. 

Its purpose is to:
- Install software or packages 
- Modify the image by running shell commands.
- Set up the environment or perform configuration tasks.

### Question : What is the purpose of the CMD command in a Dockerfile?
The CMD command in a Dockerfile specifies the default command that will run when a container starts from the built image.

## Docker Container Lifecycle
- A process is a running program in a system that can have child processes or threads.
- A container functions similarly to a process, with states like started, stopped, and killed.

![alt text]({{ site.baseurl }}/assets/postimg/life.png)
- Containers can be created (creation state), started (running state), paused (pause state), stopped (stopped state), and deleted (end state).

![alt text]({{ site.baseurl }}/assets/postimg/state.png)

Commands like 
- ``docker run`` or ``docker create`` create containers
- ``docker start`` runs them  
- ``docker pause`` pauses them 
- ``docker stop`` stops them  
- ``docker rm`` deletes them.

- Containers live inside the Docker engine and can be managed using Docker commands.
- Managing container states helps control resource usage and container availability on the system

### Question : What are the different states in the Docker container lifecycle?
The different states in the Docker container lifecycle are:
- Created: The container is created but not yet running.
- Running: The container is actively running.
- Paused: The container is temporarily paused, suspending its processes.
- Stopped: The container is stopped and not running.
- Deleted (or Killed): The container is removed from the system and no longer available.

## Working with Containers

```Dockerfile
docker run <container name/id> # running the container 

docker ps # listing the running container 

docker pause < container name/id> # pausing the running container

docker unpause <container name/id> # unpausing the container 

docker restart <container name/id> # restarting the stopped container

docker stop <container name/id> # stop the running container

docker kill <container name/id> # forcefully kill a container 

docker rmi <container name/id> # removing container

docker ps -a # confirming container are removed

```
